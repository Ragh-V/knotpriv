# Surgeries and Double Branched Covers – README

This project studies when the **double branched cover** of a knot in \(S^3\) can also be described as a **Dehn surgery** on a (possibly different) knot. The strategy is:

1. Use **Heegaard Floer homology** to rule out most slopes via an *HF rank bound*.
2. On the remaining slopes, compare **Casson invariants** of the double branched cover and the surgery manifolds.
3. For the still-remaining slopes, compare **Turaev–Viro invariants** of the same manifolds.

The code is organized as a small Python package, `surgeries_dbc`, plus several Sage “driver” scripts that orchestrate the full pipeline inside a Sage-enabled Docker container. There is also an optional workflow to run the Turaev–Viro step in **native Python with Regina + SnapPy** outside Docker.

This document is written for someone who is comfortable with Python, Docker, and basic 3-manifold notions.

---

## 0. Layout and core scripts

**Package**

- `surgeries_dbc/`
  - `models.py` – defines `KnotRecord` (name, DT codes, invariants, HF data, etc.).
  - `io.py` – all file I/O:
    - load knot lists, DT codes, numeric DT codes
    - write/read classical invariant lists
    - write Casson logs and `failList.txt`.
  - `invariants_det_only.py` – computation of **determinants only**.
  - `invariants.py` – full classical invariants:
    \(\det(K), A(K), \sigma(K), J_K(t), \lambda(\Sigma_2(K))\).
  - `hf_casson.py` – HF data attachment + original HF + Casson comparison.
  - `hf_filter.py` – **HF-only filter**, using det + HF rank bounds to eliminate slopes before Casson.
  - `casson_only.py` – Casson comparison given precomputed classical data.
  - `tv_compare.py` – Turaev–Viro comparison for slopes listed in `failList.txt`.

**Driver scripts (Sage)**

- `run_pipeline.sage`  
  Original 4-stage pipeline that:
  1. Builds numeric DT list.
  2. Computes **all** classical invariants for all knots.
  3. Runs HF + Casson comparison, writing:
     - `Casson-comparison-results.txt`
     - `failList.txt` (slopes not ruled out).
  4. Runs Turaev–Viro comparison on slopes in `failList.txt`.

- `run_fast_pipeline.sage`  
  **HF-first “fast” pipeline:**
  1. Build numeric DT list if needed.
  2. Load knots.
  3. Compute **determinants** (with caching via `detList.txt`).
  4. Attach HF data (`mList`, `nList`, `HFboundList`).
  5. Run HF-only filter → `hf_failList.txt` + a set of **HF-surviving slopes**.
  6. Compute *classical invariants only for HF-survivor knots*, with caching via
     `aList.txt`, `sigList.txt`, `jonesList.txt`, `cassonList.txt`.
  7. Run Casson comparison on HF-survivors → final `failList.txt`.
  8. Run Turaev–Viro comparison on slopes in `failList.txt`.

- `run_det_pipeline.sage`  
  Minimal pipeline to compute **only** `detList.txt` from DT data.

- `run_casson_only.sage`  
  Casson-only pipeline that assumes classical lists already exist and:
  1. Attaches det + HF data.
  2. Runs HF-only filter (HF survivors only).
  3. Loads classical invariants **only for HF-survivor knots** from disk.
  4. Runs Casson comparison on those slopes, writing:
     - `Casson-comparison-results.txt`
     - `failList.txt`.

These scripts all assume the data directory is mounted as `/work` and the repo as `/repo` inside the Docker container.

## 1. Required data files in `/work`

At minimum, the following text files (one entry per line) are expected in `/work`:

- `knotList.txt` – knot names (e.g. `11n42`, `12n345`).
- `DTList.txt` – alpha DT codes, one per knot.
- `mList.txt` – integer \(m\) data for HF obstruction.
- `nList.txt` – integer \(n\) data for HF obstruction.
- `HFboundList.txt` – HF rank bound data.

Optional (possibly cached) lists:

- `numDTList.txt` – numeric DT codes (this is usually generated by `DT-convert.sage`).
- `detList.txt` – determinants \(\det(K)\).
- `aList.txt` – `A(K)` values, stored as SymPy-readable strings.
- `sigList.txt` – signatures \(\sigma(K)\).
- `jonesList.txt` – Jones polynomials (SymPy expressions).
- `cassonList.txt` – Casson invariants of double branched covers \(\lambda(\Sigma_2(K))\).

The pipelines will **reuse** existing lists when possible and compute only what is missing.

## 2. Running pipelines in Docker

Everything is designed to run inside a Sage-enabled Docker image, e.g.:

```bash
docker run --platform=linux/amd64 -it \
  -v "$PWD":/work \
  -v "$(cd ..; pwd)":/repo \
  computop/sage:latest \
  sage /repo/run_fast_pipeline.sage
```

Here:

- `"$PWD"` is the data directory containing `knotList.txt`, `DTList.txt`, etc.  
- `"$(cd ..; pwd)"` is the parent directory containing the repo (with `surgeries_dbc/` and the driver scripts).

### 2.1. Original full pipeline

Use when you want the original “compute everything” behavior:

```bash
docker run --platform=linux/amd64 -it \
  -v "$PWD":/work \
  -v "$(cd ..; pwd)":/repo \
  computop/sage:latest \
  sage /repo/run_pipeline.sage
```

This will:

1. Create `numDTList.txt` if missing.
2. Compute and/or load all classical invariants.
3. Run HF + Casson comparison and write `Casson-comparison-results.txt` and `failList.txt`.
4. Run Turaev–Viro comparison using `failList.txt`.

### 2.2. Fast HF-first pipeline

Use when you want to avoid computing classical invariants for knots that HF already rules out:

```bash
docker run --platform=linux/amd64 -it \
  -v "$PWD":/work \
  -v "$(cd ..; pwd)":/repo \
  computop/sage:latest \
  sage /repo/run_fast_pipeline.sage
```

Typical behavior:

- If `detList.txt` exists: load `det(K)` from file.  
  Otherwise: compute all determinants and write `detList.txt`.

- Attach HF data and run the HF rank obstruction.  
  This writes `hf_failList.txt` and gives a dictionary of **HF-surviving slopes**.

- Classical invariants are then computed **only** for those knots whose slopes survive HF, and cached per knot index via `aList.txt`, `sigList.txt`, `jonesList.txt`, `cassonList.txt`.

- Casson comparison runs only on HF-survivors, and the final slopes that Casson cannot distinguish are written to `failList.txt` in the format:

  ```text
  14n1009; DT:[...]; p = 13; q = [-5, -4, -3, -2, -1, 1, 2, 3, 4]
  ```

- Finally, Turaev–Viro comparison is run on the slopes in `failList.txt`.

If at some point you already have a trustworthy `failList.txt` and only want to redo Turaev–Viro, it is safe to run the **TV stage directly** using `tv_compare.py` (see §4).

### 2.3. Determinant-only pipeline

To only build `detList.txt`:

```bash
docker run --platform=linux/amd64 -it \
  -v "$PWD":/work \
  -v "$(cd ..; pwd)":/repo \
  computop/sage:latest \
  sage /repo/run_det_pipeline.sage
```

This:

- Uses DT codes and SnapPy to compute \(|\Delta_K(-1)|\) for every knot.
- Writes `detList.txt` with one integer per knot.

### 2.4. Casson-only pipeline (with precomputed classical data)

This is useful when:

- You already have `detList.txt`, `aList.txt`, `sigList.txt`, `jonesList.txt`, `cassonList.txt` from a previous experiment / external computation.
- You want to re-run **only** the HF + Casson logic to produce a fresh `failList.txt`.

Run:

```bash
docker run --platform=linux/amd64 -it \
  -v "$PWD":/work \
  -v "$(cd ..; pwd)":/repo \
  computop/sage:latest \
  sage /repo/run_casson_only.sage
```

This will:

1. Load the knot records.
2. Load determinants from `detList.txt`.
3. Attach HF data and run the HF-only filter → `hf_failList.txt` + HF-survivors.
4. Load \(A(K)\), \(\sigma(K)\), Jones, and Casson(DBC) **only for HF-survivor knots** from the existing lists (with progress reporting).
5. Run Casson comparison on the surviving slopes and write:
   - `Casson-comparison-results.txt`
   - `failList.txt` (post-Casson survivors only).

No classical recomputation is done; everything is read from disk.

## 3. Optional external Turaev–Viro workflow (native Python + Regina)

For performance reasons, you may want to run Turaev–Viro invariants **outside Docker**, using your local installs of **SnapPy** and **Regina**. The code supports this via the `tv_compare` module.

### 3.1. Python virtual environment

Create an environment (example using `python3.11`):

```bash
python3.11 -m venv tv-env
source tv-env/bin/activate
pip install snappy-manifolds regina-normal
pip install sympy
```

Make sure this succeeds:

```bash
python -c "import snappy, regina, sympy; print('ok')"
```

### 3.2. Running Turaev–Viro comparison on a directory

Pick a working directory that already contains a `failList.txt` produced by one of the pipelines (Docker or otherwise). Then, from the repo root:

```bash
source tv-env/bin/activate   # if not already activated
python -m surgeries_dbc.tv_compare /path/to/working-directory
```

This will:

- Read `failList.txt` in that directory.
- For each line \(K; DT; p; \{q\}\):
  - Build the double branched cover and surgery manifolds using SnapPy.
  - Convert to Regina triangulations.
  - Compute Turaev–Viro invariant at level 5 using Regina’s `turaevViro(5, alg=regina.ALG_TREEWIDTH)`.
  - Compare values and update:
    - `whatsLeft.txt` – remaining \(q\)-values that Turaev–Viro has not ruled out.
    - `exceptions.txt` – cases where Turaev–Viro fails to distinguish.

The script is resumable: it uses `whatsLeft.txt` and `progress.txt` to know what’s already been done. If it is killed partway through, you can re-run the same command and it will pick up where it left off (modulo the time-limit logic for slow covers).

## 4. Output files and their meaning

Key text files written into `/work` (or the external TV directory):

- `detList.txt` – determinants \(\det(K)\).
- `aList.txt`, `sigList.txt`, `jonesList.txt` – classical invariants.
- `cassonList.txt` – Casson invariants of double branched covers \(\lambda(\Sigma_2(K))\).
- `hf_failList.txt` – slopes not ruled out by HF rank bounds.
- `Casson-comparison-results.txt` – detailed HF + Casson comparison log, listing for each slope:
  - Casson(DBC)
  - Casson(surgery)
  - whether Casson distinguishes or not.
- `failList.txt` – the “working list” of slopes **not ruled out** by Casson (used as input for Turaev–Viro step).
- `whatsLeft.txt` – Turaev–Viro’s internal working list of remaining slopes (updated as slopes are ruled out).
- `exceptions.txt` – slopes for which Turaev–Viro does not distinguish the branched cover from the surgery (the genuinely troublesome cases).
- `progress.txt` – textual progress log for Turaev–Viro.

## 5. Suggested usage patterns

- **Exploratory work / small datasets**  
  Use `run_pipeline.sage` to produce everything (classical + HF + Casson + TV) in one go.

- **Large datasets**  
  Use `run_det_pipeline.sage` once to build `detList.txt`.  
  Then use `run_fast_pipeline.sage` to:
  - apply HF obstruction first,
  - compute classical data only where needed,
  - and hand a trimmed `failList.txt` to Turaev–Viro.

- **Re-running Casson with updated HF bounds / lists**  
  Use `run_casson_only.sage` with precomputed `aList.txt`, `sigList.txt`, `jonesList.txt`, `cassonList.txt`.

- **Heavy TV work**  
  Once you have a `failList.txt`, either:
  - let `run_fast_pipeline.sage` drive the TV stage inside Docker, or  
  - switch to the external `python -m surgeries_dbc.tv_compare ...` workflow with native SnapPy + Regina.

---

That should be consistent with the code you’re currently running; if you rename any scripts or decide to drop the “original” pipeline entirely in favor of the fast one, it’s easy to thin this down.
